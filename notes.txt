# Git & GitHub Learning Notes

## 1. What is Git?

### Definition

Git is a distributed version control system used to track changes in files and collaborate with other developers.

### Why Git?

* Tracks file changes
* Maintains history
* Enables collaboration
* Supports branching and merging
* Allows rollback to previous versions

---

## 2. What is GitHub?

### Definition

GitHub is a cloud platform that hosts Git repositories.

### Workflow

```text
Local Repository
       ↓
     Push
       ↓
    GitHub
       ↑
     Pull
       ↑
Other Developers
```

### Common Uses

* Store code online
* Team collaboration
* Backup projects
* Open-source contributions

---

# 3. Git Repository

A repository (repo) is a project tracked by Git.

### Initialize Repository

```bash
git init
```

### Example

```bash
mkdir Developer-Main
cd Developer-Main
git init
```

Output:

```text
Initialized empty Git repository
```

---

# 4. Git Status

Shows current repository state.

### Command

```bash
git status
```

### Uses

* Check modified files
* Check staged files
* Check current branch

### Example

```text
On branch main
nothing to commit
working tree clean
```

---

Creates a snapshot of changes.


Shows commit history.

### Command


# 8. Git Remote

Connects local repository to GitHub.

### Add Remote

```bash

```

### Example

```bash

```

### Verify

```bash

```

---

# 9. Git Push

Uploads commits to GitHub.

### Command


### Example

```text
      ↓
```

# 12. Git Restore

Discards local changes.

### Single File

```bash
git restore index.html
```

### All Files

```bash
git restore .
```

### Example

Before:

```html
<h1>Wrong Change</h1>
```

After:

```bash
git restore index.html
```

Returns to last committed version.

# 13. Branches

## What is a Branch?

A branch is an independent line of development.

### Main Branch

```text
main
  |
  A
```

### Feature Branch

```text
main
  |
  A

feature
  |
  A
```

---

## Create Branch

```bash
git branch feature
```

---

## View Branches

```bash
git branch
```

Output:

```text
* main
  feature
```

---

## Switch Branch

```bash
git switch feature
```

---

## Create + Switch

```bash
git switch -c feature
```

---

## Merge Branch

```bash
git switch main
git merge feature
```

### Before Merge

```text
main
  A

feature
  A → B
```

### After Merge

```text
main
  A → B

feature
  A → B
```

---

## Delete Branch

```bash
git branch -d feature
```

---

# 14. Local Branch vs Remote Branch

### Local Branch

Exists only on your machine.

```text
Developer-Main
└── feature
```

### Remote Branch

Exists on GitHub.

Push it:

```bash
git push -u origin feature
```

Now GitHub shows:

```text
main
feature
```

# 15. Errors Encountered and Solutions

### Error 1

```text
src refspec master does not match any
```

### Reason

Current branch was main.

### Solution

```bash
git push origin main
```

---

### Error 2

```text
remote contains work that you do not have locally
```

### Reason

GitHub had commits missing locally.

### Solution

```bash
git pull origin main
git push origin main
```

---

### Error 3

```text
fatal: not a git repository
```

### Reason

Not inside a Git repository.

### Solution

```bash
cd project-folder
git status
```
